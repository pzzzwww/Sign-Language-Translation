# 手语识别系统 — 云服务器部署指南

## 整体流程

```
本地开发 → 代码推 Git → 服务器拉代码 → 装依赖 → 启动服务 → 外网访问
```

## 服务器配置

| 项目 | 最低 | 推荐 |
|------|------|------|
| CPU | 2核 | 4核 |
| 内存 | 4G | 8G |
| 系统盘 | 40G | 60G |
| 系统 | Alibaba Linux / CentOS 8 / Ubuntu 22.04 | — |

实测 2核4G 可以跑，内存占用约 1G（Qwen2-0.5B + MediaPipe + FastAPI）。

## 部署步骤

### 1. 登录服务器

```bash
ssh root@你的服务器IP
```

### 2. 创建 Python 环境

```bash
conda create -n torch python=3.10 -y
conda activate torch
```

### 3. 拉取项目代码

```bash
git clone 你的仓库地址 ~/Transformer
cd ~/Transformer
```

### 4. 安装依赖

```bash
pip install -r requirements.txt
```

这一步最久，torch 和 transformers 都很大，等几分钟。

### 5. 测试启动

```bash
python -m src.backend.main
```

看到 `Uvicorn running on https://0.0.0.0:8000` 就说明成功了。`Ctrl+C` 停掉。

### 6. 配置 systemd（开机自启 + 进程守护）

创建服务文件 `/etc/systemd/system/sign-language.service`：

```ini
[Unit]
Description=SL Sign Language System
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/Transformer
Environment=PATH=/opt/conda/envs/torch/bin:/usr/sbin:/usr/bin
ExecStart=/opt/conda/envs/torch/bin/python -m src.backend.main
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

然后激活：

```bash
systemctl daemon-reload
systemctl enable sign-language
systemctl start sign-language
```

### 7. 开放安全组端口

阿里云控制台 → ECS → 安全组 → 入方向添加规则：

| 端口 | 协议 | 授权对象 |
|------|------|----------|
| 8000 | TCP | 0.0.0.0/0 |

### 8. 访问

浏览器打开 `https://服务器IP:8000`，浏览器提示证书不安全时点"高级 → 继续访问"。

---

## 日常管理

```bash
systemctl status sign-language   # 看状态
systemctl stop sign-language     # 停止
systemctl start sign-language    # 启动
systemctl restart sign-language  # 重启
journalctl -u sign-language -f   # 看实时日志
```

## 面试可能会问的

### Q: 为什么用 HTTPS 而不是 HTTP？

A: 浏览器安全策略要求 HTTPS 才能调用摄像头。项目使用自签名证书，Nginx 反向代理时可以换成 Let's Encrypt 正式证书。

### Q: 服务器重启了怎么办？

A: 配置了 systemd，`enable` 后开机自动启动，不用手动管。`Restart=on-failure` 还能在进程崩溃时自动拉起来。

### Q: 模型文件怎么传到服务器的？

A: CSL 模型（13MB）和 LoRA 适配器直接放在 Git 仓库里。Qwen2 基础模型（~1GB）通过 HuggingFace 缓存目录随代码一起上传。

### Q: 内存够用吗？

A: 实测 4G 服务器跑完整链路（MediaPipe + CSL Transformer + Qwen2 LoRA + pyttsx3 TTS）内存占用约 1G，剩余充足。

### Q: 有多少人可以用？

A: 单实例适合演示和个人使用。多个用户同时访问会因为 WebSocket 和 GPU 推理排队变慢。生产环境可以加 Nginx 负载均衡 + 多实例。

### Q: systemd 和 nohup 有什么区别？

A: nohup 只是让进程在后台跑，终端关了不会停。systemd 是系统级的服务管理器，能做更多：开机自启、进程挂了自动重启、统一管理日志、控制启动顺序等。生产环境都用 systemd。
