## 二、学术界 vs 本项目

学术界的顶尖方案（SLRT、Uni-Sign、SpaMo、MMSLT）共同特征是：

- 在 CSL-Daily 等公开 benchmark 上追求 SOTA 分数
- 只做识别这一个环节
- 无前端界面，无语音输出，不考虑部署
- 依赖多 GPU 推理，普通电脑跑不动

**本项目的差异化**：

> 学术界造发动机，本项目造整车。

| | 学术界方案 | 本项目 |
|---|-----------|--------|
| 目标 | 发论文，刷 benchmark | **能用的产品** |
| 用户 | 审稿人 | **任何人** |
| 硬件 | 多 GPU 集群 | **普通笔记本 CPU** |
| 输出 | 识别准确率数字 | **听到的语音** |

## 五、为什么这个项目稀缺？

### 1. 中文手语本身就是小众方向

全球手语研究 90% 集中在 ASL（美国手语）和 DGS（德国手语）。CSL（中国手语）数据少、研究少、工具少。2025 年 Uni-Sign 发布的 CSL-News 数据集是第一个大规模中文手语数据集。

### 2. 端到端工程化极其罕见

学术圈只管"识别准确率"这一个数字。把识别、翻译、TTS、前端、WebSocket、部署全串起来的工程化项目，全球开源社区几乎找不到第二个。

### 3. CPU 实时推理是硬门槛

大模型方案（Uni-Sign、MMSLT）精度高但需要 GPU，无法部署到普通电脑。本项目 2.8M 参数的轻量 Transformer 在 CPU 上实时推理，这是**有意为之的设计选择**，不是能力不够。

### 4. 面试/简历角度的稀缺性

- 涵盖 CV（MediaPipe + ViT）+ NLP（Qwen2 + LoRA）+ 系统设计（FastAPI + WebSocket）+ 工程化（降级、训练管线）
- 随便拿出一个模块都够讲 10 分钟
- 面试官大概率没见过同类项目
